# dbt_demo — Pack B PoC against Snowflake

End-to-end dbt project that consumes the 5 Pack B mock seeds and (next iteration) the 10 Phase 5 specs to produce ready-to-demo data products on Snowflake.

## Layout

```
dbt_demo/
├── dbt_project.yml          project config (profile: dbt_demo)
├── profiles.yml.example     copy to ~/.dbt/profiles.yml and fill in
├── seeds/
│   ├── _seeds.yml           per-seed Snowflake column types + descriptions
│   ├── ll_quote_setup.csv
│   ├── ll_quote_policy_detail.csv
│   ├── quote_policy_detail.csv
│   ├── ll_quote_coverage_detail.csv
│   ├── rate_monitoring.csv  (layer-grain rate change)
│   └── rate_monitoring_total_our_share_usd.csv  (quote-grain roll-up)
├── models/                  empty for now — staging + marts go here
├── macros/  tests/  analyses/  snapshots/
└── .gitignore               target/, dbt_packages/, profiles.yml
```

## Schemas created in Snowflake

| Layer | Schema | Built by |
|-------|--------|----------|
| Raw | `<database>.raw_pack_b` | `dbt seed` |
| Staging | `<database>.stg_pack_b` | `dbt run --select staging` (views) |
| Marts | `<database>.mart_pack_b` | `dbt run --select marts` (tables) |

## Setup

```bash
# 1. Install dbt-snowflake (one-time, in any venv)
pip install dbt-snowflake

# 2. Set up the Snowflake profile
mkdir -p ~/.dbt
cp dbt_demo/profiles.yml.example ~/.dbt/profiles.yml
# edit ~/.dbt/profiles.yml — fill in account, user, role, database, warehouse

# 3. From inside dbt_demo/
cd dbt_demo
dbt debug             # verifies the Snowflake connection
dbt seed              # loads the 5 CSVs into raw_pack_b
```

After `dbt seed` you should have these tables in `<database>.raw_pack_b`:

| Table | Rows | Grain |
|---|---|---|
| `ll_quote_setup` | 250 | quote_id |
| `ll_quote_policy_detail` | 393 | (quote_id, layer_id, pas_id) |
| `quote_policy_detail` | 393 | same as ll_quote_policy_detail |
| `ll_quote_coverage_detail` | 520 | (quote_id, layer_id, pas_id, coverage_id) |
| `rate_monitoring` | ~280 | (quote_id, layer_id) (renewals only) |
| `rate_monitoring_total_our_share_usd` | 182 | quote_id (renewals only — premium-weighted aggregation of `rate_monitoring`) |

The two rate-monitoring seeds reconcile by construction: the quote-grain
seed is built by **premium-weighted aggregation** of the layer-grain seed,
weighted by each layer's `tech_gnwp_full`. Use the layer-grain seed for
per-layer adequacy / attribution work; use the quote-grain seed for
quote-level prioritisation.

## Refreshing the seeds

The CSVs are generated by `scripts/generate_mock_pack_b.py` at the repo root, which writes directly into `dbt_demo/seeds/`. To regenerate with a different seed or row count, edit constants in that script and re-run:

```bash
.venv/Scripts/python.exe scripts/generate_mock_pack_b.py
dbt seed --full-refresh
```
